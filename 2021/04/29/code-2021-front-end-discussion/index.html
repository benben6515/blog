    2021 前端領航者論壇

    發表於 2021-04-29 分類於 程式筆記 Disqus：
    日期 : Apr. 28 20:10 - 22:20 | 人數 : 240 - 250 | 系統 : Zoom

    議題一 : 解決前端知識焦慮者的所有困惑

    • 為何現在前端人才搶手？
    • 非本科就比較弱勢嘛？
    • 什麼人適合當前端當工程師？
    • 不同學習方式 ( 線上、實體 ) 的差異？
    • 前端門檻低，是否容易過飽和？

    議題二 : 從各面向探討解決派系選擇障礙

    • 所以框架要選 Vue, React, Angular ？
    • 我應該要學 TypeScript 嗎？
    • 應該不斷的追求新技術嗎？

    議題三 : 初階工程師想知道的答案

    • 我的績效是怎樣算的？
    • 我改怎麼做才能加薪、升遷？

    議題四 : 老司機的告白，告訴你真實的玩法

    • 找工作該如何證明自己？
    • 業界面試有無大原則可循？
    • 產業環境對職涯發展有多大的影響？
    • 過往精進自我的方式？

    筆記

    • 議題一 :

      • 前端的人很多，但能打的很少。
      • 太依賴框架，基本能力、原理了解不深。
      • 薪水比較優渥，所以很有熱忱(?)。
      • 要有熱忱，真的喜歡前端，假日看 code 、下班寫 code 寫到不想回家。
      • 有入門經驗之後，要盡量靠自己去學習，經驗不是單靠課程可以跟上的。
      • 大量的練習、feedback 是必要的。
      • 前端的熱忱才是重要的，非本科系、本科系 ( 約 3-5 成 ) 差別不大，解決能力才是關鍵。
      • 挑 senoir 工程師看技術；挑 junior 工程師看態度。
      • 前端難民(?) : 上了 2 - 3 月的課，但是課程的範例不會修改，沒有學到精隨。
      • 建議 : 不要拿課程內容當作品集。

    • 議題二 :

      • 框架比較 : Vue 簡單好上手；Recat 居多、薪水比較高(?)；Angular 被排擠QQ
      • 學哪個框架都可以，現在差別不大了，只是要先學哪個的問題，主要看當前公司需求。
      • TypeScript 越來越多公司導入，可能是未來趨勢。
      • TypeScript 一開始用會不習慣，習慣後開發方便。
      • 如果 JavaScript 學的好， TypeScript 幫助有限，TypeScript 主要幫助 - 型別問題。
      • 最近的新玩意 : Svelte
      • 要先思考新東西哪裡可以用到，先 run 幾個小專案，團隊都 OK 就可以試試看。

    • 議題三 :

      • 要如何分辨 senoir、junior，如果覺得自己有到，就丟履歷，哪裡不足就去補。
      • senoir 會考 JavaScript 核心知識、 code review …等。
      • 打開 github 看寫幾行 code 評斷 KPI (?)
      • 有沒有辦法達成任務、有沒有符合時程、…等。
      • 新人有問題不要怕問，一個人卡在那，可能會影響公司的進度。
      • 記錄自己的績效、做了甚麼 ( evidence ) ，之後才可以跟主管報告，爭取加薪、升遷。

    • 議題四 :

      • 你的作品集、專案，遇到什麼困難、你的思路是甚麼。
      • 身為前端工程師多或少都要有個 side project 在 github 上很重要，可以看出寫的程式碼、思路。
      • 有的公司面試資深，可能不會去考太多，因為怕會被反問。
      • 大概面試流程 : 可能會有前測，演算法、回家作業等，之後看你的 code 討論技術類問題，最後跟 HR 面試，中級主管面試等。
      • 你這兩年遇到最難的問題是甚麼？可以隨手寫筆記紀錄思路。
      • 如果想拿國外 offer 推薦大家可以刷 leetcode ，至少 300 題。
      • 職涯剛開始比較辛苦， know how 很重要。
      • 要思考如何把程式跟其他產業結合 ( 傳產、接案、博藝…等 )。
      • 做產品，技術要深；做接案，技術要廣。
      • 轉職完工程師，要保持著學習 Never End 的精神。(Front-End = Never-End)
      • 保持精進做好兩件事 : 1.工作上的技術要加深熟練 2.新技術要廣度的搜尋了解
      • 小黃鴨除錯法 : 除錯或測試過程中，操作人耐心地向小黃鴨解釋每一行程式的作用，以此來激發靈感與發現矛盾。
